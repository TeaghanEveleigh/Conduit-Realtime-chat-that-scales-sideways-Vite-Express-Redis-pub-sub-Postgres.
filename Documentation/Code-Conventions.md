# Code Conventions 

## Purpose  

The purpose of this file is to illustrate all the code conventions and style chooses leveraged throughout the app, The aim for this is to allow for a consistent and clean structure that is highly readable. The most important thing I have focused on in this code is to enforce a common naming convention as much as possible.

# Backend Decisions

- **Folder Structure**: Feature Based 
- **API Structure** Using a Feature Based structure where each endpoint is its own confined module, decided to go with this methodology as I feel it is more clean and keeps everything confined where it logically should be




# Frontend Decisions

- **React Componenets** in PascalCase
- **File & Folder names** in kebab-case
- **Functions & Variables** in camelCase includes hooks etc
